Bu Workshop Nedir?

~30 dk

30 dakika boyunca canlı, beraber, sıfırdan bir Telegram bot kuracağız. Import yok — her node'u kendin oluşturacaksın. Sonunda kendi n8n instance'ında Türkçe SQL sorabilen bir botun olacak.

▶ Sonunda elinde olacak

Telegram'dan "en çok satan 5 müzik türü" yazınca Markdown tablo dönen kendi botun. Chat geçmişi hatırlanan, hatadan kendini düzelten, salt-okunur SQL üreten bir agent.

▶ Adımlar (yüksek seviye)
›BotFather → ›n8n credentials → ›Telegram Trigger → ›AI Agent → ›2 HTTP Tool → ›System Prompt → ›Telegram Send → ›Test.
▶ Önkoşullar

Telegram hesabı · n8n erişimi (self-hosted veya Cloud) · OpenRouter API key (free tier yeterli).

02 /

Chinook Veritabanı Nedir?

SQLite · D1

Chinook, dijital bir müzik mağazasını modelleyen örnek bir SQLite veritabanıdır. Workshop için 2021-2025 arası sipariş geçmişi yüklendi — ~3500 parça, 11 tablo, gerçekçi ama anlaşılır boyut. Sıfırdan SQL öğrenirken kullanılan standart dataset'lerden biri.

ArtistSanatçı kayıtları (Iron Maiden, Metallica, ...)
AlbumAlbümler, her biri bir sanatçıya bağlı
TrackParçalar (album, genre, süre, fiyat)
GenreMüzik türleri (Rock, Latin, Metal, ...)
MediaTypeFormat (MP3, AAC, Protected MPEG-4, ...)
CustomerMüşteri kayıtları, ülke + email
InvoiceFatura başlıkları (toplam tutar, tarih, ülke)
InvoiceLineFatura kalemleri (parça + adet + birim fiyat)
EmployeeÇalışanlar (manager hiyerarşili)
PlaylistÇalma listeleri
PlaylistTrackListe-parça eşlemesi (many-to-many)

Tam ER diyagramı: data/chinook-schema.mmd (Mermaid kaynak — VS Code/Mermaid Live'da aç)

Cloudflare Worker arka planda ne yapıyor?

edge · D1 · HTTP

n8n bir SQLite dosyasına direkt erişemez — araya bir HTTP API koymamız lazım. Bu işi Cloudflare Worker yapıyor: dünya çapında düşük latency, free tier'a sığar, D1 (yönetilen SQLite) ile entegre. Worker iki endpoint sunuyor — biri güvenli önizleme için, diğeri tam sonuç için.

Endpoint Ne yapar Body Dönüş
POST /test SQL'i LIMIT 5 ile sarar — güvenli önizleme {"sql": "..."} {"rows": [...]} veya {"error": "..."}
POST /execute SQL'i olduğu gibi çalıştırır {"sql": "..."} {"rows": [...]} veya {"error": "..."}
$ Worker base URL
https://chinook-workshop.bitter-brook-7999.workers.dev
Güvenlik Worker src/security.ts regex'i INSERT/UPDATE/DELETE/DROP/ALTER/CREATE/REPLACE/ATTACH/DETACH/PRAGMA/VACUUM kelimelerini engelliyor. Sadece SELECT geçer. Salt-okunur garanti prompt'ta değil, Worker sınırına gömülü.
$ Worker sağlık testi (terminalde çalıştır)
curl -s -X POST \ -H "Content-Type: application/json" \ -H "User-Agent: curl/8.0.0" \ -d '{"sql":"SELECT 1 AS health"}' \ https://chinook-workshop.bitter-brook-7999.workers.dev/test

Beklenen: {"rows":[{"health":1}],"count":1}

Adım 1 — Telegram Bot oluştur (BotFather)

Telegram · ~2 dk
01 BotFather: /newbot + token al ~ 2 dk

Telegram'da @BotFather ile sohbete başla. /newbot komutuyla yeni bot oluştur. İstediği bot adı + kullanıcı adını ver (kullanıcı adı _bot ile bitmeli). Sonunda HTTP API token'ı verir — bunu güvenli bir yere kopyala.

$ BotFather komutu (Telegram'a yapıştır)
/newbot
Beklenen sonuç BotFather sana "Use this token to access the HTTP API:" diye başlayan bir mesaj döner — token bu satırın altındadır.
06 /

Adım 2 — n8n credentials kur

n8n UI · ~3 dk
02 Credentials: Telegram API + OpenRouter API ~ 3 dk

n8n UI → Credentials → New. İki credential ekleyeceğiz: Telegram Bot Token + OpenRouter API Key.

  • Telegram credential: type seç → Telegram API → "Access Token" alanına BotFather'dan aldığın token'ı yapıştır → Save.
  • OpenRouter credential: type seç → OpenRouter API → "API Key" alanına openrouter.ai/keys'den aldığın key'i yapıştır → Save.
Beklenen sonuç Credentials listesinde iki kayıt — yeşil tick (✓) işaretli.
06 /

Adım 3 — Yeni workflow + Telegram Trigger ekle

n8n canvas · ~3 dk
03 Workflow + Telegram Trigger node ~ 3 dk

Yeni boş workflow aç. Sol panelden Telegram Trigger node'unu canvas'a sürükle. Az önce oluşturduğun Telegram credential'ı seç. Updates listesinden Message işaretle. Kaydet.

Beklenen sonuç Canvas'ta yalnız bir Telegram Trigger node'u, webhook URL'i n8n tarafından üretilmiş halde.
06 /

Adım 4 — AI Agent node ekle (Tools Agent mode)

n8n canvas · ~3 dk
04 AI Agent node — Tools Agent ~ 3 dk

Telegram Trigger'ın yanına AI Agent node ekle (sol panelden "AI Agent" ara, sürükle). Aşağıdaki ayarları yap:

$ Agent type (UI dropdown)
Tools Agent
$ Prompt Type (UI dropdown)
Define
$ Prompt (text alanı)
={{ $json.message.text }}
Beklenen sonuç AI Agent canvas'ta, Trigger'a bağlı, kırmızı uyarı yok (System Message + tools sonra eklenecek).
06 /

Adım 5 — OpenRouter Chat Model sub-node bağla

sub-node · ~2 dk
05 OpenRouter Chat Model — claude-haiku-4.5 ~ 2 dk

AI Agent'ın altındaki "+" işaretine tıkla → "Chat Model" → "OpenRouter Chat Model" seç. Az önce oluşturduğun OpenRouter credential'ı seç. Model alanına şunu yapıştır:

$ Model — default (kararlı + hızlı + ucuz)
anthropic/claude-haiku-4.5

Alternatif modeller aşağıdaki "Model Alternatifleri" bölümünde — istediğin zaman değiştir.

Beklenen sonuç OpenRouter Chat Model sub-node AI Agent'a bağlı, model adı görünür.
06 /

Adım 6 — Window Buffer Memory sub-node bağla

sub-node · ~2 dk
06 Window Buffer Memory — Custom Key ~ 2 dk

Yine "+" → "Memory" → "Window Buffer Memory" seç. Session ID type'ı "Custom Key" yap. Session Key alanına şunu yapıştır:

$ Session Key — her Telegram chat'i ayrı geçmiş
={{ $('Telegram Trigger').item.json.message.chat.id }}

Bu sayede aynı chat'teki önceki sorular hatırlanır, farklı chat'ler izole kalır.

Beklenen sonuç Memory sub-node AI Agent'a bağlı, custom session key set.
06 /

Adım 4 — Tool 1 ekle

HTTP Tool · ~4 dk
07 Tool 1: generate_and_test_sql ~ 4 dk

AI Agent'a ilk HTTP tool'unu ekle. Bu tool üretilen SQL sorgusunu LIMIT 5 ile kuru-çalıştırır (dry-run): yanıt başarılıysa AI'a güveni gelir, hata varsa AI sorguyu düzeltip tekrar dener. Önce her zaman bu çağrılır, sonra execute_sql.

$ Tool Description (toolDescription alanına yapıştır)
Validates a SQL query by running it with LIMIT 5 against the Chinook SQLite backend. Returns {rows:[...]} on success, {error:'...'} on failure. ALWAYS call this before execute_sql.
$ Endpoint URL (URL alanına yapıştır)
https://chinook-workshop.bitter-brook-7999.workers.dev/test
$ Body parameter — name: sql, value:
={{ $fromAI('sql', 'The SQL SELECT query to dry-run with LIMIT 5. SQLite syntax only.', 'string') }}
